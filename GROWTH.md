# Nonley - User Acquisition Strategy

## Phase 1: Launch Week (Day 1-7)

### 1. Product Hunt Launch

- **When**: Tuesday or Wednesday (highest traffic days)
- **Prep**: Create a 60-second demo video showing the extension in action
- **Tagline**: "See who else is browsing the same page as you, right now"
- **What to show**: Install extension > go to YouTube > see others > send friend request > chat
- **Hunter**: Find an established hunter on Twitter/X to post for you (higher visibility)
- **Launch day**: Respond to every comment within 30 minutes

### 2. Hacker News (Show HN)

- Post as "Show HN: Nonley - The presence layer of the internet"
- Lead with the technical angle: SHA-256 URL hashing for privacy, WebRTC peer-to-peer calls, zero-dep embed
- Be ready to answer security/privacy questions in detail
- Best time: 6-8 AM EST on a weekday

### 3. Reddit Launch Posts

- **r/SideProject** - "I built a Chrome extension that shows who else is on the same webpage as you"
- **r/InternetIsBeautiful** - Focus on the serendipity angle
- **r/chrome** - "Chrome extension for seeing co-presence on any website"
- **r/webdev** - Technical breakdown of how the presence engine works
- **r/startups** - Startup journey + product story
- Do NOT spam all subreddits on the same day. Space over 1-2 weeks.

### 4. Indie Hackers

- Post in the milestones section
- Write a short article about building it
- Engage in the community before posting

## Phase 2: Community Seeding (Week 2-4)

### 5. Target Specific Communities

The product works best when people are on the SAME page. Seed specific communities:

**YouTube Communities**

- Post in YouTube creator Discord servers
- "See who else is watching the same video right now"
- Partner with small-medium YouTubers (10k-100k subs) to mention it

**Reddit Communities**

- Each subreddit is a community. If 50 people from r/programming install Nonley,
  they'll start seeing each other on the same Stack Overflow pages
- Target interest-based subreddits with high engagement

**Twitch / Gaming**

- Viewers watching the same stream can connect
- Gaming communities browse the same wikis/guides
- Partner with small Twitch streamers

**Online Learning**

- Students on the same Coursera/Udemy course page
- Study groups forming organically
- Post in r/learnprogramming, r/datascience, r/OMSCS

### 6. Embed Widget for Website Owners

- Reach out to blog owners, forum operators, community sites
- Offer the free embed widget: "Show your visitors who else is reading right now"
- Target Substack writers, Ghost blog owners, indie sites
- The embed creates a flywheel: visitors see Nonley > install extension > use everywhere

## Phase 3: Growth Loops (Month 2-3)

### 7. The Invite Loop

- When a user sees someone on a page and sends a friend request,
  the other person MUST have Nonley to accept
- Add a "Share Nonley" button in the extension popup
- "You saw 3 people today but couldn't connect - invite friends to grow your network"

### 8. The Website Badge

- Create an embeddable "Live on Nonley" badge for websites
- Like the old "Powered by WordPress" but it says "3 people here right now"
- Clicking it installs the extension
- Website owners get the badge for free; it drives extension installs

### 9. Content Marketing

- Write blog posts about:
  - "The loneliest corners of the internet" (data from anonymous presence counts)
  - "What 10,000 co-browsing sessions taught us about the internet"
  - "Why the internet feels lonely and how we're fixing it"
- Post on Medium, Dev.to, personal blog
- SEO: Target "browse together", "who is online", "internet presence"

### 10. Twitter/X Strategy

- Daily posts showing interesting presence patterns
- "Right now, 47 people are reading the same Wikipedia article about black holes"
- Build in public: share user counts, technical decisions, milestones
- Engage with indie hacker / startup community

## The Cold Start Problem

Nonley has a classic cold start problem: it's only useful when others are using it.

**Solutions:**

1. **Focus on niches first**: Get 100 users who all browse the same sites (e.g., a specific subreddit's community) rather than 1,000 random users
2. **Events**: "Nonley Watch Party" - coordinate 50 people to watch the same YouTube video at the same time. Share the experience on social media.
3. **Launch day surge**: Coordinate all your launch channels (PH, HN, Reddit) within a 48-hour window to create density
4. **University pilot**: Get 200 students at one school to install it. They'll see each other on course pages, library sites, etc.
